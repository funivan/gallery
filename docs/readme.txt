# Технічна сторона:

- Objects
  Справжній обєкт визначається в першу чергу поведінкою. А пізніше уже даними.
  В теорії нам має бути все рівно на те як обєкт працює з даними, він має виконувати певну роботу.
  Хороший приклад це ResponseCookieInterface Для фреймворку головне це підставити дані cookies в response
  Як ви будете їх ініціалізувати - це не його проблема.
  Також є невдалі обєкти - FinderFilter
  Невдалі обєкти зафіксовані міткою:
    take a look: Data transfer object


- Immutable.
  Дуже специфічна парадигма. Коли починав писати - був її прихильником.
  Зараз - не впевнений що вона взагалі потрібна.
